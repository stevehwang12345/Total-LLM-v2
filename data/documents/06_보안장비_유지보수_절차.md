# 보안 장비 유지보수 및 점검 절차서

## 1. 목적
본 절차서는 물리보안 장비의 예방 정비, 장애 대응, 예비 부품 관리에 대한 기준을 정의한다. ISO 27001:2022 제어 7.13(장비 유지보수) 및 ISMS-P 2.4.4(보호설비 운영)에 근거한다.

## 2. 점검 주기별 항목

### 2.1 일일 점검 (매일 09:00)
| 점검 대상 | 점검 항목 | 합격 기준 | 담당 |
|----------|---------|---------|------|
| CCTV | 전체 카메라 영상 출력 확인 | 모든 카메라 정상 영상 출력 | 관제 요원 |
| CCTV | NVR 녹화 상태 확인 | 녹화 정상 진행 중 | 관제 요원 |
| 출입통제 | 주요 출입문 개폐 동작 확인 | 정상 잠금/해제 | 경비 요원 |
| 알람 패널 | 시스템 상태 표시등 확인 | 정상(Green) 표시 | 관제 요원 |
| UPS | 전원 상태 표시등 확인 | 정상 충전 상태 | 시설 관리 |

### 2.2 주간 점검 (매주 월요일)

|----------|---------|---------|

## 3. 장애 대응 절차

### 3.1 장애 등급 및 목표 복구 시간
| 등급 | 영향 범위 | 목표 복구 시간 (RTO) | 예시 |
|------|---------|------------------|------|
| **P1 (긴급)** | 통제구역 보안 기능 상실 | 2시간 이내 | 서버실 CCTV 전체 장애, 출입통제 마비 |
| **P2 (높음)** | 주요 구역 보안 기능 저하 | 8시간 이내 | NVR 1대 장애, 일부 센서 오작동 |
| **P3 (보통)** | 일반 구역 단일 장비 장애 | 24시간 이내 | 단일 카메라 장애, 비주요 센서 |

### 3.2 장애 대응 프로세스
```
1. 장애 접수 (상황실 또는 자동 감지)
    ↓
2. 등급 판정 (P1/P2/P3)
    ↓
3. 긴급 조치 (예비 장비 투입, 인력 순찰 강화)
    ↓
4. 원인 분석 (하드웨어/소프트웨어/네트워크/환경)
    ↓
5. 수리 또는 교체
    ↓
6. 복구 검증 (정상 동작 확인)
    ↓
7. 장애 보고서 작성 + 재발 방지 대책
```

## 4. 예비 부품 관리

### 4.1 최소 재고 기준
| 장비 | 최소 예비 수량 | 비고 |
|------|-------------|------|
| IP 카메라 (동일 모델) | 전체의 5% | 신속 교체 |
| PoE 스위치 | 1대 | 네트워크 장애 대비 |
| NVR 하드디스크 | 2개 | RAID 복구 |
| 출입통제 리더기 | 2개 | 고장 즉시 교체 |
| UPS 배터리 | 1세트 | 수명 도래 대비 |

### 4.2 관리 원칙
- 예비 부품은 온습도가 관리되는 별도 보관함에 저장한다.
- 유통기한(배터리) 및 호환성(펌웨어 버전)을 분기별 확인한다.
- 반출입 기록을 유지하며, 사용 시 즉시 재구매 요청한다.

## 5. 관련 법규 및 표준
- ISO 27001:2022 Annex A 7.13 (장비 유지보수)
- ISMS-P 2.4.4 (보호설비 운영)
- 전기사업법 (UPS, 비상발전기 점검 의무)
- 소방시설법 (소화설비 점검 의무)
